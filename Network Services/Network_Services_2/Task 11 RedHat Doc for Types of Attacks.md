| Exploit                             | Description                                                                                                   | Notes                                                                                                                                                                                                                                                                   |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Null or Default Passwords**       | Leaving administrative passwords blank or using vendor-supplied default passwords.                            | - Common in routers, firewalls, VPNs, and NAS devices.- Also found in legacy operating systems that bundle services.- Administrators sometimes create privileged accounts and leave passwords blank, creating an easy entry point for attackers.                        |
| **Default Shared Keys**             | Using default cryptographic keys intended for testing or development in production environments.              | - Common in wireless access points and secure appliances.- If unchanged, anyone with the default key can access the protected resource.- Example: CIPE ships with a sample static key that must be changed before deployment.                                           |
| **IP Spoofing**                     | An attacker impersonates a trusted host on the network to exploit vulnerabilities and gain access.            | - Requires predicting TCP/IP sequence numbers.- Tools exist to aid in performing this attack.- Depends on insecure source-based authentication services such as rsh, telnet, and FTP.- SSH and SSL/TLS mitigate this risk.                                              |
| **Eavesdropping**                   | Capturing network traffic between two communicating hosts to obtain sensitive information.                    | - Most effective against plaintext protocols such as Telnet, FTP, and HTTP.- Usually requires access to a compromised machine on the LAN.- Prevented through encryption, one-time passwords, and secure authentication mechanisms.                                      |
| **Service Vulnerabilities**         | Exploiting flaws in network services to gain unauthorized access or execute arbitrary code.                   | - Common in Internet-facing services such as HTTP/CGI.- Vulnerabilities may allow remote command execution or shell access.- Buffer overflows can provide complete system compromise.- Services should avoid running as root and be kept updated with security patches. |
| **Application Vulnerabilities**     | Exploiting weaknesses in desktop or workstation applications to execute malicious code or compromise systems. | - Frequently targets email clients and user applications.- Can lead to trojan installation or further network compromise.- Users should avoid unauthorized software and suspicious attachments.- Automatic updates help reduce risk.                                    |
| **Denial of Service (DoS) Attacks** | Overwhelming a system or network resource with malicious traffic, making it unavailable to legitimate users.  | - Often performed using multiple compromised systems (botnets/zombies).- Source IP addresses are frequently spoofed.- Famous example: the large-scale attacks of 2000.- Mitigations include ingress filtering (RFC 2267), firewalls, and IDS/IPS solutions like Snort.  |
